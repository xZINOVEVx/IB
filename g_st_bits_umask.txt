1.Установка группового бита (можно ставить на файлы и на папки)
sudo chmod g+s <имя папки>
sudo chmod g+s script. bash

установка g-бита на файле позволяет запускать файл от имени группы
sudo chmod g+s Fold1

Установка g-бита на папку позволяет наследовать группу владельцев на все подпапки и файлы, расположенные внутри
Смена группы владельцев у папки Fold1
sudo chwon :group1 Fold1
mkdir Fold1/Fold_test - создает папку с группой владельцев group1 и наследует g-бит от родительской папки
touch Fold1/Fold_test.txt - Создает файл с группой владельцев group1


Пример прав доступа к файлу с g-битом
2664/rw-rwSr--

Пример прав доступа к папке с g-битом
2775/rwxrwsr-x

Установка sticky-бита (на папки)
sudo chmod o+t <имя папки>
По умолчанию файлы, созданные владельцем внутри папки, могут удалять либо владельцы, либо члены группы владельцов
Папки с установленным t-битом запрещают удаление фалов всем, кроме владельцев
При этом при разрешении на запись можно удалить любую информацию, находящуюся внутри такого файла
sudo chmod o+t Fold3
touch Fold3/test_tbit.txt
stat Fold3/test_tbit.txt
0664/rw-rw-r-- UID student/1000 GID group1/1001
newgrp group1
cat > test_tbit.txt
Файл доступен для записи 
rm test_tbit.txt 
Сообщение от команды rm:
rm: cannot remove 'test_tbit.txt' Permission denied


пример прав доступа к папке с t-битом
1775/rwxrwx-t

При одновременной установке g и t битов на папку, наследование распространяется только на g-бит
sudo chmod go+st Fold4
stat Fold4
3775/rwxrwsr-t

mkdir Fold4/Test_gt
stat Fold4/Test_gt
2775/rwxrwsr-o
